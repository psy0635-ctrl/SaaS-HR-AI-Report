# 데이터셋 사용할 거 있나 조사하기

> 파일명: `dataset.md`

## 1. 개념

**데이터셋(dataset)**은 분석이나 AI 학습에 사용할 수 있도록 모아 놓은 데이터 묶음이다.

예를 들어 HR AI Report 기능에서는 직원 정보, 근태 기록, 성과 평가, 이직 여부, 만족도 조사 결과 등이 데이터셋이 될 수 있다.

| 데이터 예시 | 설명 |
|---|---|
| 직원 정보 | 이름, 부서, 직무, 입사일, 직급 |
| 근태 데이터 | 출근, 퇴근, 지각, 결근, 연차 사용 |
| 성과 데이터 | 평가 점수, 목표 달성률, 프로젝트 기여도 |
| 이직 데이터 | 퇴사 여부, 퇴사 시점, 이직 위험도 |
| 만족도 데이터 | 조직 만족도, 업무 만족도, 설문 응답 |

## 2. 왜 필요한가?

AI Report는 아무 데이터 없이 생성될 수 없다. AI가 의미 있는 리포트를 만들려면 **분석할 HR 데이터**가 필요하다.

우리 프로젝트에서는 데이터셋이 다음 역할을 한다.

| 역할 | 설명 |
|---|---|
| 대시보드 지표 생성 | 직원 수, 이직률, 결근률, 성과 평균 계산 |
| AI 리포트 생성 | 데이터를 요약하고 인사이트를 문장으로 생성 |
| 이직 위험 분석 | 특정 조건과 패턴을 기반으로 위험 직원 탐지 |
| 테스트 데이터 | 실제 기업 데이터 없이 기능 개발 가능 |
| 모델 검증 | AI 리포트 결과가 타당한지 실험 가능 |

## 3. 핵심 용어 정리

| 용어 | 설명 |
|---|---|
| 데이터셋 | 분석이나 학습을 위해 모아 놓은 데이터 |
| CSV | 쉼표로 구분된 표 형태 파일 |
| JSON | 객체 구조로 데이터를 표현하는 파일 형식 |
| Excel | 사람이 보기 좋은 표 계산 파일 형식 |
| 더미 데이터 | 실제 데이터가 부족할 때 테스트용으로 만든 가짜 데이터 |
| 라벨 | 예측 대상 값. 예: 퇴사 여부 |
| 컬럼 | 표 데이터의 항목. 예: 나이, 부서, 급여 |
| 행 | 한 명의 직원 또는 하나의 기록 단위 |
| 개인정보 | 개인을 식별할 수 있는 이름, 이메일, 전화번호 등 |

## 4. 동작 흐름 또는 구조

```text
데이터 수집
→ 데이터 정리
→ 개인정보 제거 또는 익명화
→ 데이터베이스 저장
→ 통계 분석
→ AI 모델 또는 LLM 입력
→ HR AI Report 생성
```

데이터 형식별 흐름은 다음과 같다.

```text
CSV / Excel / JSON
→ 백엔드 업로드
→ 데이터 검증
→ DB 저장
→ 분석 쿼리 실행
→ AI 리포트 생성
```

## 5. 사용 예시

우리 프로젝트에서 사용할 수 있는 HR 데이터는 다음과 같다.

| 데이터 종류 | 주요 컬럼 예시 | 활용 예시 |
|---|---|---|
| 직원 정보 | employee_id, department, job_role, hire_date | 부서별 인원 현황 |
| 근태 데이터 | date, check_in, check_out, late_count, absence_count | 지각/결근 추세 분석 |
| 성과 데이터 | performance_score, goal_rate, manager_score | 고성과자/저성과자 분석 |
| 이직 데이터 | attrition, years_at_company, salary, overtime | 이직 위험 예측 |
| 만족도 데이터 | job_satisfaction, work_life_balance, survey_score | 조직 만족도 리포트 |

추천 데이터셋 예시:

| 출처 | 데이터셋 | 활용 방향 |
|---|---|---|
| Kaggle | IBM HR Analytics Employee Attrition & Performance | 이직 위험 분석, 성과 분석 |
| Kaggle | HR Analytics 관련 공개 데이터 | 직원 이탈, 승진, 만족도 분석 |
| 공공데이터포털 | 고용, 노동, 산업 관련 공공데이터 | 외부 통계 참고 |
| KOSIS | 고용률, 임금, 산업별 근로 통계 | 대시보드 비교 지표 |
| AI Hub | 한국어 문서, 상담, 요약 관련 데이터 | HR 챗봇, 문서 요약 테스트 |

특히 **IBM HR Analytics Employee Attrition & Performance 데이터셋**은 직원 이직 여부, 나이, 부서, 직무, 만족도, 성과 등 HR 분석에 필요한 컬럼이 포함되어 있어 팀 프로젝트용으로 적합하다.

근태 데이터가 부족하다면 다음처럼 **더미 데이터**로 보완할 수 있다.

| employee_id | date | check_in | check_out | late_count | absence |
|---|---|---|---|---|---|
| E001 | 2026-05-01 | 09:03 | 18:02 | 1 | false |
| E002 | 2026-05-01 | 08:55 | 18:10 | 0 | false |
| E003 | 2026-05-01 | - | - | 0 | true |

## 6. 장점

- 실제 서비스와 비슷한 HR 분석 기능을 만들 수 있다.
- AI Report 결과가 더 구체적이고 설득력 있게 나온다.
- 대시보드, 차트, 리포트 기능을 테스트하기 쉽다.
- 실제 개인정보 없이도 더미 데이터로 개발할 수 있다.
- 공개 데이터셋을 사용하면 초기 개발 속도가 빨라진다.

## 7. 주의할 점

- **실제 직원 개인정보를 그대로 사용하면 안 된다.**
- 데이터셋마다 라이선스가 다르므로 사용 가능 범위를 확인해야 한다.
- Kaggle 데이터는 학습용으로 좋지만 실제 기업 데이터와 차이가 있을 수 있다.
- 결측치, 이상치, 중복 데이터가 있을 수 있으므로 정제가 필요하다.
- 더미 데이터는 실제 분석 결과처럼 과신하면 안 된다.

## 8. 우리 프로젝트 적용 정리

우리 프로젝트에서는 공개 데이터셋과 더미 데이터를 함께 사용하는 방향이 적절하다.

| 적용 항목 | 적용 방향 |
|---|---|
| 기본 분석 데이터 | IBM HR Analytics Employee Attrition & Performance 활용 |
| 근태 데이터 | 부족한 부분은 더미 데이터 생성 |
| 만족도 데이터 | 설문 형식의 가상 데이터 추가 |
| 저장 형식 | 초기에는 CSV, 이후 DB 테이블로 전환 |
| 개인정보 보호 | 이름, 이메일 대신 employee_id 사용 |
| AI Report 입력 | 통계 요약 결과와 주요 패턴만 AI에 전달 |
