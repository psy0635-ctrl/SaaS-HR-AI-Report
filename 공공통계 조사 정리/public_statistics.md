# 공공통계 조사 정리

## 1. 조사 목적

이 문서는 AI 기반 HR 리포트 MVP에서 사용할 더미 HR 데이터를 현실적인 범위로 만들기 위해 작성한다.  
Kaggle 데이터셋은 컬럼 구조를 참고하는 용도이고, 공공통계는 근로시간, 임금, 입직률, 이직률, 고용형태 같은 수치 범위를 정하는 기준으로 사용한다.

조사일: 2026-05-20  
적용 대상: `employees`, `attendance`, `performance`, `attrition` 더미 데이터 생성 규칙

## 4. 더미 데이터 생성에 반영할 추천 범위

### 4.1 직원 수와 부서 구성

MVP에서는 너무 큰 데이터보다 화면과 리포트가 잘 보이는 규모가 중요하다.

| 항목 | 추천값 |
|---|---|
| 전체 직원 수 | 300~500명 |
| 부서 수 | 5개 |
| 분석 기간 | 최근 6개월 |
| 주요 부서 | 개발, 영업, 고객지원, 마케팅, 인사 |

부서 비율 예시:

| 부서 | 비율 |
|---|---:|
| 개발팀 | 30% |
| 영업팀 | 25% |
| 고객지원팀 | 20% |
| 마케팅팀 | 15% |
| 인사팀 | 10% |

---

### 4.2 근태 데이터

공공통계의 월평균 근로시간은 전체 근로시간 기준이다.  
더미 데이터에서는 일별 출근/퇴근 기록을 만들고, 월 단위로 합산했을 때 자연스러운 범위가 나오도록 설정한다.

| 항목 | 추천 생성 규칙 |
|---|---|
| 출근 시간 | 08:45~09:20 사이 |
| 퇴근 시간 | 18:00~20:30 사이 |
| 월 총 근로시간 | 140~175시간 중심 |
| 월 초과근무 시간 | 일반 직원 0~20시간, 고위험 직원 20~45시간 |
| 지각률 | 전체 근무일의 3~8% |
| 결근률 | 전체 근무일의 1~4% |

AI 리포트에서 활용할 수 있는 문장 예시:

```text
이번 달 개발팀의 평균 초과근무 시간이 전월보다 증가했으며,
일부 직원은 월 30시간 이상의 초과근무를 기록해 이직 위험 관리가 필요합니다.
```

---

### 4.3 임금 데이터

공공통계상 2025년 연간 월평균 임금총액은 약 420만원 수준이므로, 전체 평균이 이 값 근처에 오도록 생성한다.  
다만 직급, 부서, 고용형태별 차이를 만들어야 분석 화면이 자연스럽다.

| 직급 | 월급 추천 범위 |
|---|---:|
| 사원 | 2,600,000~3,600,000원 |
| 대리 | 3,300,000~4,500,000원 |
| 과장 | 4,200,000~5,800,000원 |
| 차장 | 5,200,000~7,000,000원 |
| 부장 | 6,500,000~9,000,000원 |

고용형태별 보정:

| 고용형태 | 급여 보정 |
|---|---|
| 정규직 | 1.00배 |
| 계약직 | 0.65~0.90배 |
| 인턴/수습 | 0.50~0.70배 |

---

### 4.4 입직률과 이직률

공공통계의 입직률/이직률은 전체 사업체 노동이동 기준이므로, SaaS HR 더미 데이터에서는 발표와 시연에 적합하도록 조금 안정적인 범위로 사용한다.

| 항목 | 추천 생성 규칙 |
|---|---|
| 최근 6개월 신규 입사자 비율 | 전체 직원의 5~10% |
| 최근 6개월 퇴사자 비율 | 전체 직원의 8~15% |
| 월별 대시보드 이직률 | 1~6% 사이에서 변동 |
| 고위험 직원 비율 | 전체 직원의 5~12% |

부서별 차이를 주면 리포트가 더 자연스럽다.

| 부서 | 이직 위험 패턴 예시 |
|---|---|
| 개발팀 | 초과근무가 많은 직원의 위험 점수 상승 |
| 영업팀 | 성과 하락과 목표 미달 직원의 위험 점수 상승 |
| 고객지원팀 | 결근/지각이 많은 직원의 위험 점수 상승 |
| 마케팅팀 | 변동 폭은 중간 수준 |
| 인사팀 | 비교적 안정적인 패턴 |

---

### 4.5 고용형태와 채용시장 데이터

공공데이터포털의 고용24 구인구직 및 취업동향 자료는 구인인원, 구직건수, 취업건수를 월별로 제공하며 지역별, 산업별, 고용형태별 조회가 가능하다.  
이 자료는 우리 회사 내부 HR 데이터라기보다 외부 채용시장 흐름을 보여주는 자료로 보는 것이 좋다.

프로젝트 적용 방법:

| 활용 위치 | 적용 방법 |
|---|---|
| 대시보드 보조 지표 | "채용시장 참고 지표" 카드로 활용 가능 |
| AI 리포트 문장 | "외부 구인구직 흐름을 고려하면..." 같은 참고 문장 생성 |
| 더미 데이터 | 신규 입사자 수나 채용 난이도 설정에 보조적으로 활용 |

주의할 점:

- 고용24 자료는 고용24를 이용한 구인, 구직 상황만 포함한다.
- 전체 노동시장을 완전히 대표하는 값으로 사용하면 안 된다.
- 내부 직원 퇴사율이나 성과 데이터로 직접 변환하지 않는다.

##  6. public_statistics.xlsx 파일 역할

이 파일은 실제 직원 데이터를 담고 있는 데이터셋이 아니라,
HR 더미 데이터를 현실적인 범위로 생성하기 위한 공공통계 기준표입니다.

활용 예시는 다음과 같습니다.

- 월 근로시간 생성 기준: 140~175시간
- 평균 월급 생성 기준: 약 420만 원
- 최근 6개월 신규 입사자 비율: 5~10%
- 최근 6개월 퇴사자 비율: 8~15%
- 계약직 급여 비율: 정규직 대비 65~90%

따라서 이 파일은 Python 더미 데이터 생성 스크립트의 기준 자료이자,
발표/보고서에서 데이터 신뢰성을 설명하는 근거 자료로 사용합니다.
