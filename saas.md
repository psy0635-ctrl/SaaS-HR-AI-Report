# SaaS 알아보기

> 파일명: `saas.md`

## 1. 개념

**SaaS(Software as a Service)**는 사용자가 프로그램을 직접 설치하지 않고, **인터넷을 통해 소프트웨어를 사용하는 방식**이다.

예를 들어 Gmail, Notion, Slack, Google Drive처럼 웹 브라우저에서 로그인만 하면 바로 사용할 수 있는 서비스가 SaaS에 해당한다.

우리 프로젝트의 “SaaS HR 플랫폼”은 기업 고객이 별도 설치 없이 웹에서 로그인하여 직원 정보, 근태, 성과, AI 리포트를 관리하는 형태를 의미한다.

| 구분 | 설명 |
|---|---|
| SaaS | 인터넷으로 접속해서 사용하는 소프트웨어 |
| HR SaaS | 인사 관리 기능을 웹 서비스 형태로 제공 |
| AI Report | HR 데이터를 분석해 AI가 리포트를 생성하는 기능 |

## 2. 왜 필요한가?

우리 프로젝트는 여러 기업이 사용할 수 있는 HR 플랫폼을 목표로 한다. 따라서 단순한 개인용 프로그램보다 **로그인, 권한 관리, 데이터 저장, 구독형 서비스 구조**가 중요하다.

| 필요한 이유 | 설명 |
|---|---|
| 여러 회사가 사용 | 회사별로 직원 데이터가 분리되어야 함 |
| 웹 기반 접근 | 설치 없이 브라우저에서 사용 가능 |
| 권한 관리 | 관리자, 인사 담당자, 일반 직원 권한 구분 필요 |
| 데이터 저장 | 직원, 근태, 성과, 리포트 데이터를 서버에 저장 |
| 구독형 서비스 | Basic, Pro, Enterprise 같은 요금제 구성 가능 |

## 3. 핵심 용어 정리

| 용어 | 설명 |
|---|---|
| SaaS | 소프트웨어를 설치하지 않고 인터넷으로 사용하는 서비스 방식 |
| 클라우드 | 서버, 저장소, 데이터베이스 등을 인터넷 기반으로 제공하는 환경 |
| 테넌트 | SaaS를 사용하는 고객 단위. 예: A회사, B회사 |
| 멀티테넌시 | 하나의 서비스가 여러 고객 데이터를 분리해 제공하는 구조 |
| 구독형 서비스 | 월/연 단위로 요금을 지불하고 서비스를 사용하는 방식 |
| 인증 | 사용자가 누구인지 확인하는 과정. 예: 로그인 |
| 인가 | 사용자가 어떤 기능에 접근할 수 있는지 확인하는 과정 |
| SLA | 서비스 제공자가 보장하는 가용성, 성능, 보안 수준 |

## 4. 동작 흐름 또는 구조

```text
사용자
→ 웹 브라우저 접속
→ 로그인
→ 권한 확인
→ HR 플랫폼 기능 사용
→ 직원/근태/성과 데이터 조회
→ AI Report 생성 요청
→ AI 분석 결과 저장
→ 리포트 화면 제공
```

SaaS HR 플랫폼의 기본 구조는 다음과 같다.

```text
기업 고객 A ┐
기업 고객 B ├→ SaaS HR 플랫폼 → 공통 백엔드 → 데이터베이스
기업 고객 C ┘                         ↓
                                  AI Report 엔진
```

중요한 점은 **기업별 데이터가 섞이면 안 된다**는 것이다. 예를 들어 A회사의 인사 담당자가 B회사의 직원 정보를 보면 심각한 보안 문제가 된다.

## 5. 사용 예시

우리 프로젝트에서 SaaS 구조를 적용하면 다음과 같은 기능을 만들 수 있다.

| 기능 | 예시 |
|---|---|
| 로그인 | HR 담당자가 이메일과 비밀번호로 로그인 |
| 권한 관리 | 관리자만 전체 직원 리포트를 볼 수 있음 |
| 직원 데이터 저장 | 직원 이름, 부서, 직무, 입사일 저장 |
| AI Report 생성 | “이번 달 이직 위험 직원 요약 리포트 생성” |
| 구독 관리 | 무료 플랜은 월 5개 리포트, 유료 플랜은 무제한 생성 |

예시 화면 흐름:

```text
관리자 로그인
→ 대시보드 접속
→ 직원 데이터 업로드
→ AI Report 생성 클릭
→ 부서별 근태/성과/이직 위험 분석
→ PDF 또는 웹 리포트로 확인
```

## 6. 장점

- **설치가 필요 없어 사용 진입 장벽이 낮다.**
- 서버에서 기능을 업데이트하면 모든 사용자가 최신 버전을 사용할 수 있다.
- 회사별로 구독 요금제를 적용할 수 있어 서비스화하기 좋다.
- 웹 기반이라 PC, 태블릿 등 다양한 환경에서 접근 가능하다.
- 데이터가 중앙 서버에 저장되어 백업과 관리가 쉽다.

## 7. 주의할 점

- **기업별 데이터 분리**가 제대로 되지 않으면 개인정보 유출 위험이 있다.
- 로그인, 비밀번호, 토큰 관리 등 보안 설계를 신중히 해야 한다.
- SaaS는 서버 장애가 나면 모든 사용자가 영향을 받으므로 안정성이 중요하다.
- 무료/유료 기능 제한을 명확히 설계해야 한다.
- HR 데이터는 민감 정보가 많으므로 접근 권한을 최소화해야 한다.

## 8. 우리 프로젝트 적용 정리

우리 프로젝트는 단순한 AI 리포트 생성기가 아니라 **여러 기업이 사용할 수 있는 SaaS형 HR 플랫폼**으로 설계하는 것이 좋다.

| 적용 항목 | 적용 방향 |
|---|---|
| 로그인 | 관리자, HR 담당자, 일반 직원 계정 구분 |
| 권한 관리 | 역할 기반 접근 제어 적용 |
| 데이터 저장 | 직원, 근태, 성과, 만족도, 리포트 테이블 구성 |
| AI Report | 회사별 HR 데이터를 기반으로 자동 리포트 생성 |
| 구독형 서비스 | 리포트 생성 횟수, 저장 기간, AI 기능 범위로 요금제 구분 |
| 보안 | 회사별 데이터 분리, 개인정보 암호화, 접근 로그 관리 |
